# Contributing Guide

## Objetivo

Este documento define as regras iniciais de contribuição para o **ConnectionCyber Engineering Framework (CCEF)**.

## Regra principal

Nenhuma contribuição deve ser feita sem respeitar os princípios do CCEF:

- Knowledge First
- Architecture First
- Customer First
- Documentation First
- Security by Design
- Everything Versioned
- Everything Traceable

## Fluxo recomendado

1. Registrar a necessidade de mudança.
2. Avaliar se a mudança exige ADR.
3. Atualizar ou criar o artefato correspondente.
4. Registrar impacto em changelog, roadmap ou journal quando necessário.
5. Submeter para revisão.
6. Aprovar antes de considerar o artefato oficial.

## Padrão de commits

Utilizar Conventional Commits:

```text
docs: atualização documental
feat: nova funcionalidade
fix: correção
chore: manutenção
refactor: reorganização
test: testes
ci: integração contínua
security: segurança
```

## Idioma

- Documentação principal: Português do Brasil.
- Código, APIs e identificadores técnicos: Inglês.
