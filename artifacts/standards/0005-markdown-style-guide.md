# STD-0005 — Markdown Style Guide

## 1. Objetivo

Definir o padrão oficial de escrita Markdown para artefatos do ConnectionCyber Engineering Framework.

## 2. Idioma

Documentação principal em Português do Brasil. Código, APIs, identificadores e nomes técnicos devem preferir inglês.

## 3. Títulos

Utilizar apenas um H1 por arquivo. Seções principais usam H2. Subseções usam H3.

## 4. Links

Usar links relativos.

## 5. Tom de escrita

Claro, técnico, direto, sem ambiguidades e orientado à execução.

## 6. Histórico

Todo artefato deve incluir tabela de histórico quando aplicável.

## 7. Dependências

- STD-0001 — Documentation & Artifact Standard
